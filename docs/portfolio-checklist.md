# Portfolio Checklist

Use this checklist to turn the repository into a stronger GitHub portfolio project.

## GitHub repo setup

- Set the repository description to: `Interactive Streamlit dashboard exploring maternal vaccination coverage among pregnant women in the US.`
- Add topics: `streamlit`, `python`, `pandas`, `data-visualization`, `public-health`, `cdc`, `render`, `dashboard`
- Pin the repo on your GitHub profile.
- Add the live demo URL to the GitHub repo website field: `https://vaccination-coverage-dashboard.onrender.com/`

## Best screenshots to capture

- `overview-desktop.png`
  Show the hero section, KPI cards, and the coverage trend chart.
- `geography-tab.png`
  Show the top geographies chart and the current slice stats.
- `equity-lens.png`
  Show subgroup variation and the confidence interval chart.
- `mobile-view.png`
  Capture the app in a narrow browser width to prove responsiveness.

## Capture style

- Use the live Render app instead of localhost so the screenshots reinforce that the project is deployed.
- Prefer a 1440px wide desktop browser for the three main screenshots.
- Keep the browser chrome minimal and crop out distractions.
- Use one thoughtful filter combination, not the untouched default state.
- Keep image names consistent so they can be dropped into the README later.

## Good filter combinations

- Vaccine: `Tdap`
- Geography Type: `States`
- Season: latest available season
- Dimension Type: `Race and Ethnicity`

Alternative:

- Vaccine: `Influenza`
- Geography Type: `States`
- Season: latest available season
- Dimension Type: `Age`

## README upgrade order

1. Add the live demo section near the top.
2. Add 2 or 3 screenshots under a `Screenshots` section.
3. Keep setup instructions below the visual story.
4. End with deployment and contribution notes.

## Optional polish

- Create a short screen recording or GIF for the overview tab.
- Add a GitHub social preview image using `docs/assets/project-banner.svg` or a PNG exported from it.
- Add a short project story in the README about the problem, the deployment challenge, and the improvement work.
